# IPAM (IP 주소 관리) 사용자 가이드

IPAM(IP Address Management) 서비스는 사내 인프라의 IPv4 주소 자원을 효율적으로 관리하고, 자동화된 스캔을 통해 IP 사용 상태를 실시간으로 모니터링할 수 있는 도구이다.

---

## 1. 리소스 계층 구조 및 사용 권한

### 리소스 계층 구조
IPAM은 물리적/논리적 관리를 위해 아래와 같은 계층 구조로 리소스를 구조화한다.
- **Location (위치)**: 인프라가 위치한 물리적/지리적 단위 (예: IDC, Office 등).
- **Network (네트워크)**: Location 내부에 정의된 논리적 그룹 단위.
- **Subnet (서브넷)**: 실제 IP 주소가 할당되고 관리가 이루어지는 CIDR 블록 단위 (IPv4, 최대 `/24` 대역 지원).
- **IP Address (IP 주소)**: 서브넷 내에 존재하는 개별 호스트 IP.

### 권한 모델
역할에 따라 제공되는 기능이 다르게 제한된다.
- **일반 사용자 (Viewer)**: IPAM 정보 조회 및 스캔 히스토리, 검색 기능 이용 가능.
- **관리자 (Admin)**: Location/Network/Subnet의 생성·수정·삭제, 수동 재스캔 실행, 개별 IP의 메타데이터(호스트네임, 설명) 편집 및 상태 강제 변경(Override) 가능.

---

## 2. IP 상태 관리 및 수동 변경 (Manual Override)

### IP 상태 정의
IP 주소는 스캔 결과 및 사용자 설정에 따라 아래의 4가지 상태 중 하나를 가진다.

| 상태 | 설명 | UI 표시 |
| :--- | :--- | :--- |
| **`used` (사용 중)** | Ping 반응이 있거나 관리자가 사용 중으로 설정한 상태 | **초록색** |
| **`reserved` (예약됨)** | 실제 사용 중은 아니나 향후 할당을 위해 미리 선점해 둔 상태 | **주황색** |
| **`offline` (오프라인)** | 과거 사용된 이력이 있으나, 최근 스캔에서 지속적으로 응답이 없는 상태 | **빨간색** |
| **`free` (사용 가능)** | 할당되지 않았으며 즉시 사용 가능한 상태 | **회색** |

### IP 예약 및 수동 상태 변경 (Manual Override)
스캔 결과에 의존하지 않고 특정 IP를 직접 제어해야 하는 경우를 위해 오버라이드 기능을 제공한다.
- **상태 오버라이드 (`used` / `reserved`)**:
  - 관리자가 IP 상세 편집창에서 상태를 `used` 또는 `reserved`로 변경하면 **Manual Override** 상태가 된다.
  - 이 경우, 자동 Ping 스캔이 돌더라도 해당 IP의 상태와 연속 실패 횟수 정보가 초기화되거나 변경되지 않고 유지된다. (단, 핑 성공 시 마지막 활성 시간인 `lastSeenAt` 타임스탬프는 갱신된다.)
- **오버라이드 해제 (`free`)**:
  - 관리자가 IP 상태를 `free`로 되돌리면 오버라이드가 해제된다.
  - 다음 자동 스캔 주기부터 ICMP Ping 결과에 따라 상태가 자동으로 자동 갱신된다.

## 3. IP 자동 스캔 및 수동 스캔

### 자동 스캔 (Auto Discovery)
시스템 백엔드에서 백그라운드로 실행되는 주기적 스캔 기능이다.
- **스캔 주기**: Subnet별로 자동 스캔 주기를 설정할 수 있다. (최소 1,800초 ~ 최대 86,400초, 기본값 3,600초)
- **동작 방식**: 
  - 설정된 주기에 따라 백엔드 Prober가 OS `ping`을 사용하여 ICMP 스캔을 수행한다.
  - 스캔 중인 서브넷은 임시 Lease(실행 권한 Lock)를 획득하여 동일 서브넷에 대한 중복 스캔을 방지한다.
- **상태 전환 규칙**:
  - **Ping 성공**: 상태가 `used`로 변경되며 연속 실패 횟수가 0으로 리셋된다.
  - **Ping 실패**: 연속 실패 횟수가 증가한다. 과거 성공 이력이 있었던 IP에 한해 **3회 연속 실패 시** `offline` 상태로 자동 전환된다. 성공 이력이 전혀 없는 IP는 계속 `free` 상태를 유지한다.

### 수동 스캔 (Rescan)
관리자가 스캔 주기와 무관하게 특정 서브넷의 IP 상태를 즉시 동기화하고 싶을 때 사용한다.
- **실행 방법**: IPAM 관리 테이블의 액션 영역에서 **Scan** 버튼을 클릭하여 실행한다.
- **동시 실행 방지**: 이미 스캔이 진행 중인 서브넷에 재스캔을 요청할 경우 `409 Conflict` 오류가 발생하며 요청이 무시된다.
- **강제 실행 허용 (15분 예외)**: 네트워크 지연 등으로 스캔이 비정상적으로 멈추는 현상을 방지하기 위해, 스캔을 시작한 지 15분이 경과한 서브넷에 대해서는 이전 실행 락을 무시하고 강제로 재스캔을 시작할 수 있다.

## 4. 편리한 부가 기능

### 다음 사용 가능한 IP 조회 (Next Available IP)
서브넷 내에서 비어있는 IP 주소를 직접 찾을 필요 없이 자동으로 할당 가능한 빈 IP 목록을 조회해주는 API 기능이다.
- **작동**: 지정한 서브넷 내에서 `free` 상태인 IP를 오름차순으로 탐색하여 반환한다.
- **조회 수**: 기본 5개에서 최대 100개까지 설정하여 가져올 수 있다. (자동 할당 스크립트나 프로비저닝 시스템 연동 시 유용하게 사용됨)

### 전역 검색 (Global IP Search)
IPAM 화면 상단 검색 창을 통해 IP 주소나 호스트네임으로 특정 IP를 빠르게 찾을 수 있다.
- **검색 방식**: 단어가 입력될 때마다 실시간으로 탐색을 수행(Debounce 적용)한다.
- **결과 화면**: 매칭되는 IP 주소, 호스트네임, 현재 상태 배지, 그리고 해당 IP가 속한 상세 경로(`Location / Network / Subnet`) 정보를 한눈에 보여준다. 해당 결과를 클릭하면 해당 서브넷 및 IP 상세 화면으로 바로 이동할 수 있다.

### 스캔 히스토리 및 이력 관리 (Scan History)
특정 서브넷에서 스캔이 진행되었을 때 발생한 상태 변화 및 결과를 기록하고 조회하는 기능이다.
- **메뉴 경로**: Sidebar -> **Scan History**
- **스캔 요약**: 최근에 진행된 모든 스캔 목록과 성공/실패 여부를 제공한다.
- **이력 상세 조회**: 목록에서 특정 행을 클릭하면 하단 아코디언 메뉴가 열리며 상세 로그를 확인할 수 있다.
  - 스캔 성공/실패 개수 통계
  - 스캔 시 발생한 오류 메시지 (실패한 스캔인 경우)
  - IP 상태 전이 목록 (예: `free` ➔ `used`, `used` ➔ `offline` 등)
- **변경 사항이 없는 경우**: 스캔 결과 중 상태 변화가 없었다면 `No status changes.` 메시지가 표시된다.

## 5. 예외 처리 및 사용 시 주의 사항 (데이터 제약)

1. **CIDR 대역 크기 제한**
   - 시스템 안정성을 위해 서브넷 대역은 **IPv4만 지원**하며, `/24` 대역보다 큰 범위(예: `/23`, `/16` 등, IP 개수 256개 초과 대역)는 등록이 제한된다.
2. **동일 위치(Location) 내 CIDR 중복 차단**
   - 동일한 Location 안에서는 소속 Network가 다르더라도 중복되거나 서로 겹치는(Overlap) CIDR 블록의 서브넷을 생성할 수 없다.
3. **서브넷 CIDR 수정 불가**
   - 한 번 생성된 서브넷의 CIDR 블록 정보는 수정할 수 없다. CIDR 대역을 변경해야 하는 경우 해당 서브넷을 삭제하고 신규 생성해야 한다.
4. **연쇄 삭제 (Cascade Delete) 유의**
   - 상위 계층인 Location 또는 Network를 삭제하면, 그 하위에 연결된 모든 Subnet 및 IP 주소 레코드(메타데이터 정보 포함)가 데이터베이스에서 연쇄적으로 삭제되므로 주의해야 한다.
5. **결과 덮어쓰기 방지 (Lease 메커니즘)**
   - 백그라운드 스캔이 오랜 시간 지연되어 최신 스캔이 이미 완료된 후에 뒤늦게 이전 스캔 결과가 도착하더라도, 최신 데이터를 덮어쓰지 않고 지연된 스캔 결과를 안전하게 스킵한다.
