# AGENTS.md

This directory is for omni project's documents. Documentation must always reflect the current state of the project.

## Documents Index

- `docs/`
    - `ARCHITECTURE.md`: System architecture design and technology stack details.
    - `PRD.md`: Product Requirements Document outlining the project goals and features.
    - `plans/`: A planning document for project feature design, implementations (covering only tasks of significant scope)..
        - Plan document naming: if the prefix is "ing", it indicates work in progress; if it is "done", it indicates the plan has been completed.
    - `resource-status.md`: Rules and logic for determining the health status of infrastructure resources and tools.
    - `features/` : Feature-specific documentation

---

## Rules
- Documents should be human-readable and highly legible.
- Focus strictly on logic principles and architectural design, rather than embedding specific function names or code elements in the documentation.
