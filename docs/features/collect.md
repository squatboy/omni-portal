# Resource Collection Logic

Kubernetes, VM, ArgoCD, GitLab, Nexus 등의 인프라 리소스 및 개발 도구의 상태 정보를 수집하는 백그라운드 콜렉터 기능

## 기능 동작 방식 및 플로우

1. **주기적 스케줄링**: Go 백그라운드 루프에서 30초마다 수집 프로세스를 실행.
2. **대상 조회**: PostgreSQL DB에서 활성화된(active) 리소스 및 외부 도구 연동 설정을 조회.
3. **병렬 수집**: Goroutines와 Channels를 활용해 개별 연동 대상의 API 또는 헬스 체크 엔드포인트에 요청 전송.
4. **데이터 가공**: 수집된 데이터(K8s 노드 사용량, Pod 상태, VM 핑 결과 등)를 표준화된 형태로 가공.
5. **메모리 캐싱**: 가공된 전체 리소스 상태를 스냅샷 형태로 인메모리 캐시에 저장하여 API 조회 시 즉각 응답.

## 기능 구현 방식

- **Go Concurrency 활용**: 다수의 외부 연동 대상에 대한 수집을 병렬로 처리하기 위해 Goroutines와 Channels를 적극적으로 사용함. 이를 통해 병목 현상을 방지하고 전체 수집 시간을 최소화함.
- **RWMutex 기반 인메모리 캐시**: 대시보드 API 요청마다 외부 API를 재호출하거나 DB를 조회하지 않도록 수집 결과를 `sync.RWMutex`로 보호되는 인메모리 캐시에 스냅샷으로 관리함.
- **장애 격리 설계**: 특정 연동 대상(예: Kubernetes API)의 네트워크 에러나 권한 부족 등이 발생하더라도 전체 수집 프로세스가 실패하지 않도록 각각의 수집을 독립된 태스크로 실행하며, 에러가 발생한 소스만 `down` 혹은 `unknown`으로 상태를 격리하고 메트릭을 빈 값으로 처리함.
- **크레덴셜 복호화 연동**: 외부 API 호출에 필요한 Bearer Token이나 인증 자격 증명은 DB에 암호화된 상태로 보관되어 있으며, 수집 직전 서버 메모리 상에서 복호화되어 안전하게 HTTPS 요청 헤더에 주입됨.
