# Login / Auth

단일 조직 대상의 인프라 포털 보호를 위해 제공되는 기본 ID/Password 기반의 사용자 인증 및 세션 관리 기능

## 기능 동작 방식 및 플로우

1. **최초 관리자 생성**: 데이터베이스(DB)에 등록된 사용자가 없을 때 최초 설정 화면을 열어 첫 admin 계정을 만들 수 있게 허용하며, 생성 이후에는 공개 회원가입을 차단함.
2. **계정 생성 및 임시 비밀번호 발급**: 이미 생성된 admin 계정이 Manage 화면을 통해 신규 사용자를 생성하고, 임시 비밀번호를 발급함.
3. **첫 로그인 및 비밀번호 변경**: 신규 사용자는 첫 로그인 후 즉시 비밀번호를 의무적으로 변경해야 서비스를 이용할 수 있음.
4. **세션 발급**: 로그인 검증이 완료되면 서버는 opaque session token을 발급하여 브라우저 쿠키로 보냄.
5. **로그아웃 및 세션 무효화**: 로그아웃 시 서버 세션 테이블에서 해당 토큰을 비활성화(revoke)하고, 브라우저 쿠키를 만료시킴.

## 기능 구현 방식

- **bcrypt 비밀번호 해싱**: 무차별 대입 공격을 완화하고 데이터 유출 시 보안을 확보하기 위해 사용자 비밀번호는 bcrypt 기반의 단방향 암호화 해시로 DB에 저장함.
- **Opaque Session Token 기반 세션**: JWT 등의 클라이언트 측 토큰 방식 대신, 난수 기반의 세션 토큰을 생성하고 DB에 토큰 해시와 만료 시각을 매핑하여 서버 측에서 수명과 권한을 직접 제어함.
- **쿠키 보안 설정**: 세션 토큰을 탈취당하는 것을 막기 위해 브라우저 쿠키에 `HttpOnly`와 `SameSite=Lax` 속성을 부여하여 XSS 및 CSRF 공격 경로를 방어함.
- **RBAC 권한 통제**: 권한 역할을 `admin`과 `viewer`로 세분화하여, Manage CRUD와 사용자 관리 API는 `admin` 역할만 통과하도록 웹 프레임워크 미들웨어 단에서 보호함.
- **보안 방어 대책**: 로그인 실패 횟수를 추적하여 과도한 접근 시 일시 차단하며, 계정 존재 유무가 유출되는 것을 차단하기 위해 로그인 성공/실패 여부와 무관하게 동일한 포맷의 보안 오류 응답을 유지함.
- **개발 Mock 모드**: 개발(Dev) 환경에 한하여 `NEXT_PUBLIC_OMNI_MOCK=true` 환경 변수를 사용해 로그인/인증을 우회함으로써 빠른 UI 개발 피드백을 지원하되, 프로덕션 빌드에는 포함되지 않게 차단함.
